Current Status

  • Browser app with retro desktop UI.
  • Stores data locally in localStorage.
  • Local-first storage with optional Supabase sync.
  • Works as a private planner per browser/device when sync is not configured.

Main Windows

  • Planner
  • Notes
  • README.txt

Planner Features

  • Today / Week / Month task buckets are separate.
  • Add task via retro + Add Task button.
  • Task form has one task field and date.
  • Task completion with checkbox and strikethrough.
  • Edit task.
  • Delete task with confirmation.
  • Status filter: All / Open / Done.
  • .ics calendar export with confirmation before download.
  • Planner desktop width reduced to 42rem.
  • Mobile task/form layout polished.

Notes Features

  • Notes are separate from tasks.
  • Add loose thoughts.
  • Search notes.
  • Filter notes by All / Today / Week / Month.
  • Compact previews.
  • Long notes open in a retro popup showing only the clicked note.
  • Popup has Copy, Done, and close button.
  • Notes also have a copy icon directly on the row.
  • Mobile popup respects window.visualViewport, so buttons stay above browser/taskbar chrome.
  • Notes window includes optional email-link sync controls.

README.txt In App

  • Generated daily summary/status view.
  • Shows today’s tasks, done tasks, overdue tasks, weekly count, and notes count.

Sync Discussion

  • Optional sync UI has been added using Supabase Auth and a Supabase table.
  • Auth approach: email magic link.
  • Each user signs in with their own email.
  • Same email on PC and phone syncs that person’s planner.
  • Different users stay separate.
  • Local fallback remains: app still works without sign-in/internet.
  • Setup still requires creating a Supabase project, running the SQL in README.md, adding redirect URLs, and pasting the URL/anon key into index.html.
